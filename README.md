# 10. HTTPとは何か調べてまとめる

## ◆ HTTPの仕組み（約200字）

WebブラウザとWebサーバ間でデータをやり取りする通信プロトコル<br>
クライアント:URLやメソッド（GET、POSTなど）を含むリクエストを送信<br>
サーバ:ステータスコードやコンテンツを含むレスポンスを返す。<br>
通信はステートレス(各リクエストが独立)<br>
Cookieやセッション、トークンなどを用いて状態管理を補完する。<br>
Webページ表示やAPI通信など、インターネット上の情報取得の基盤を担う。

## ◆ HTTPの特徴（約200字）

HTTPはステートレスなテキストベースのプロトコルで、１回のリクエストとレスポンスで完結する仕組みが特徴である。拡張性が高く、ヘッダにより機能追加が容易で、Webブラウザやサーバ間で広く採用されている。また、TCP上で動作し信頼性の高い接続を行う。一方、ステートレスであるため、ユーザ状態はCookieやセッション管理に依存する。安全性を高めるためにHTTPSとしてTLS暗号化と組み合わせて利用することが一般的である。

## ◆ HTTP/1.1・HTTP/2・HTTP/3の概要（約200字）

HTTP/1.1は最も普及した従来方式で、Keep-Aliveにより複数リクエストを扱うが、同時並列処理に制約がある。HTTP/2はバイナリフレームと多重化により、１つの接続で複数データを同時転送し、ヘッダ圧縮で高速化を実現する。HTTP/3はUDPベースのQUICを利用し、接続再確立なしの高速通信やモバイル環境での安定性に優れる。現在はHTTP/2が主流、HTTP/3が急速に普及している。
